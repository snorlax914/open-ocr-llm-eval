# 감자빵 1주차 보고서 - OCR

## 1. OCR 모델 성능 평가 기준

**OmniDocBench v1.5**는 2025년 CVPR에서 발표된 이후, 최신 문서 이해 모델(VLM)들의 성능을 가늠하는 핵심 벤치마크이다. 이 벤치마크 순위를 기준으로 후보군을 선정하였다.

2026년 상반기 기준으로 집계된 최신 순위와 주요 모델의 성적은 다음과 같다.

### OmniDocBench v1.5 OCR/Parsing 성능 순위 (2026년 상반기 기준)

| 순위 | 모델명 | 정확도 (Accuracy) | 비고 |
| :--- | :--- | :--- | :--- |
| **1위** | **GLM-OCR** | **94.6%** | **현재 SOTA(최고 성능)**. GPT-5.2 및 Gemini 3 Pro 능가 |
| **2위** | **PaddleOCR-VL-1.5** | **94.5%** | 바이두 오픈소스, 한국어/영어 및 복잡한 레이아웃 강점 |
| **3위** | **DeepSeek-OCR2** | **93.2%** (추정) | 효율적인 아키텍처로 빠른 추론 속도와 고성능 병행 |
| **4위** | **Gemini 3 Pro** | **90.3%** | 구글의 최신 프런티어 모델 (상업용) |
| **5위** | **MinerU-2.5** | **90.67%** | 오픈소스 PDF 추출 및 파싱 특화 모델 |
| **6위** | **Qwen3-VL (235B)** | **89.15%** | 알리바바의 대규모 시각 언어 모델 |

## 2. 주요 모델별 특징 분석 및 모델 선정

1. **GLM-OCR (94.6%)**
    * **특징:** 현재 벤치마크 1위로, 단순 텍스트 인식을 넘어 표(Table), 수식, 레이아웃 분석에서 압도적인 성능을 보인다.
    * **강점:** Markdown 출력 정합성이 매우 높으며, 기존 모델들보다 추론 처리량(Throughput)이 약 50% 개선되었다.

2. **PaddleOCR-VL-1.5 (94.5%)**
    * **특징:** 오픈소스임에도 불구하고 상용 모델인 Gemini나 GPT 시리즈를 앞서는 성적을 기록 중이다.
    * **강점:** 다국어 지원이 강력하며, 스마트폰으로 찍은 사진이나 구겨진 문서 등 실세계 노이즈에 매우 강하다.

3. **Gemini 3 Pro (90.3%)**
    * **특징:** 범용 멀티모달 모델 중에서는 최상위권이나, OCR 전용으로 튜닝된 GLM이나 PaddleOCR 계열보다는 소폭 낮은 점수를 기록했다.

이중에서 범용 멀티모달 모델(Gemini 등)은 제외하고 OCR 전용 모델 중에서 OmniDocBench v1.5 기준 SOTA 모델인 **GLM-OCR(94.6%)**, **PaddleOCR-VL-1.5(94.5%)** 과 **DeepSeek-OCR-2** 모델을 벤치마킹 대상으로 선정하였다.

## 3. 평가지표

### **편집 거리 기반 정확도(CER/WER)**
 * 공식: Error Rate = (S + D + I) / N, S: 치환(Substitution), D: 삭제(Deletion), I: 삽입(Insertion), N: 전체 글자(또는 단어) 수  

문자(Character)나 단어(Word) 단위의 오차를 계산하는 방식으로, 점수가 낮을수록 성능이 좋다.

### **소음(노이즈) 견디기**   
* Blur (흐릿함): 사진 초점이 맞지 않는 경우  
* Digits/Signs (낙서): 글자 위에 임의의 숫자나 기호가 겹쳐진 경우  
* Rotation (회전): 문서가 기울어진 경우  
* Salt and Pepper (점잡음): 스캔 품질이 낮아 검은 점들이 박힌 경우

종이가 구겨졌거나, 사진이 흐릿하거나, 낙서가 되어 있어도 글자를 잘 찾아내는지 확인한다.

노이즈의 강도(Severity)를 1단계부터 5단계까지 높여가며, 노이즈가 심해질수록 CER(글자 정확도)이 어떻게 변하는지 그래프로 측정한다.  
  
### **표 및 수식 이해도 (TEDS - Tree Edit Distance-based Similarity)**
* 공식: {TEDS}(T_a, T_b) = 1 - {{EditDist}(T_a, T_b)} over {max(|T_a|, |T_b|)}, T_a, T_b: 정답 표와 예측 표의 트리 구조  

표의 HTML 구조를 트리(Tree)로 변환하여 정답과 얼마나 일치하는지 측정한다.


### **정확 일치율(Exact Match Rate)**
* 공식:정확한 일치율(EMR) = (올바르게 추출된 필드 수/총 필드 수) * 100  
* 구조화된 데이터 추출이 필요한 사용 사례에서 OCR 정확도를 평가하는 데 중요한 지표
* 송장, 영수증 또는 신분증과 같은 표준화된 양식에서 데이터를 추출할 때 특히 유용  

이 지표는 OCR 출력 결과 가 실제 데이터와 불일치 없이 정확히 일치하는 문서 내 필드의 비율을 측정  
  
### +) **Field-Level Accuracy**
특정 필드(예: 이름, 날짜, 계좌 번호)를 정확하게 인식해야 하는 정형 및 반정형 문서에 특히 중요하다.

## 5. 진행 상황

### 5.1. Github repository 생성
Github repository를 생성하여 협업 환경을 구축한 후 팀원 간 협업 프로세스를 공유하였다.

파일 종류 별 디렉토리 구조를 정리하여 회의록, 보고서, 팀원별 조사 내용을 공유하고 모델별 테스트 코드를 작성하였다.

> Github repository 링크: https://github.com/snorlax914/open-ocr-llm-eval

### 5.2. 테스트 환경 구축
모든 모델은 huggingFace의 공식 문서를 바탕으로 테스트 환경을 구축하였다.

테스트 환경은 Githug repository에 명시되어 있는 환경을 따른다. (Python3.12 / 기타 패키지)

각 모델별 테스트 진행 상황은 아래와 같다.
#### GLM-OCR
> huggingFace 링크: https://huggingface.co/zai-org/GLM-OCR

❌ 해당 모델은 테스트 과정에서 패키지 오류가 발생하여 테스트 보류 중이다.

#### PaddleOCR-VL-1.5
> huggingFace 링크: https://huggingface.co/PaddlePaddle/PaddleOCR-VL-1.5

✅ 최신 버전 패키지 구동 오류가 발생하여 구버전으로 테스트를 완료하였다.

현재는 CLI 환경에서만 정상적으로 작동함을 확인하였다.

#### DeepSeek-OCR-2
> huggingFace 링크: https://huggingface.co/deepseek-ai/DeepSeek-OCR-2

❌ 테스트 코드를 작성하였으나 아직 테스트는 진행하지 않았다.

## 6. 향후 진행 방향

### 6.1. 모델별 가상환경 구축

최신 OCR 모델들은 의존하는 패키지 및 소프트웨어(CUDA) 버전에 민감하다. 

이로 인하여 여러 모델을 하나의 가상환경에서 돌리는 경우 패키지 충돌(**torch, transformers** 등)이 빈번하게 발생함을 확인하였다.

따라서 모델별 가상환경을 구축하여 테스트를 진행 할 예정이다.

### 6.2. 테스트 파이프라인 구축

데이터셋의 입력, 모델 테스트 코드 실행, 결과물 저장의 과정이 통합된 파이프라인을 구축하여,

스크립트(.sh 파일)를 통해 모델 성능 평가를 진행할 수 있는 효율적인 테스트 환경을 구축할 예정이다.

### 6.3. 평가 결과 시각화

파이썬의 데이터 시각화 라이브러리를 활용하여 모델 평가 결과를 시각적으로 구현할 예정이다.
