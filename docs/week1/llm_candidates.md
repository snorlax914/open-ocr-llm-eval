## 3. LLM 후보 모델 목록

본 프로젝트의 파이프라인(OCR → sLLM 분류/요약/필드 추출/검증)에 적합한  
오픈소스 경량 LLM 후보를 조사하여 정리한다.  
또한, OCR + LLM 2단계 파이프라인의 대안으로 **VLM(Vision-Language Model) 단일 모델 접근**도  
비교 후보에 포함한다.

조사 기준일: 2026년 4월 13일

### 3.2 VLM(Vision-Language Model) 후보 (2개)

VLM은 이미지를 직접 입력받아 텍스트 인식과 문서 이해를 동시에 수행한다.  
기존 파이프라인(이미지 → OCR → LLM → 결과)을 **이미지 → VLM → 결과**로 단축할 수 있어,  
OCR 단계의 오류 전파를 원천 차단하는 대안 접근이다.

#### VLM 접근의 장점
- OCR 오류가 LLM으로 전파되는 문제 자체가 사라짐
- 표, 레이아웃 등 시각 구조를 직접 이해 → 구조 보존 능력 우수
- 별도 OCR 모델 설치/관리 불필요, 파이프라인 단순화

#### VLM 접근의 단점
- 동일 파라미터 대비 텍스트 전용 LLM보다 추론 속도 느림 (이미지 처리 오버헤드)
- VRAM 요구량이 더 높음
- 단순·정형 문서에서는 전통 OCR + LLM이 더 효율적일 수 있음

---

#### 1) Qwen3-VL-4B (Alibaba) — VLM 주력 후보

| 모델 | 파라미터 | 출시 | 컨텍스트 | 라이선스 |
|---|---|---|---|---|
| Qwen3-VL-4B | 4B | 2025.10 | 256K | Apache 2.0 |

- 문서 이미지에서 직접 텍스트 인식 + Markdown 형식 출력이 네이티브 동작
- 256K 컨텍스트로 장문 문서 처리 가능
- 텍스트 전용 Qwen3.5-4B와 동일 문서로 비교 시 "OCR+LLM vs VLM" 실험이 깔끔하게 구성됨
- 출력 형식이 Markdown일 경우 VLM의 자연스러운 출력 방식과 일치하여 적합도 높음

**Colab 실행**: 무료 T4 가능

### 3.4 실험 구조

#### 파이프라인 A: OCR + 텍스트 LLM (4개)

| 순위 | 모델 | 역할 |
|---|---|---|
| 1 | **Qwen3.5-4B** | 범용 주력 후보 — 한국어 + 구조화 출력 + 컨텍스트 균형 최우수 |
| 2 | **EXAONE 3.5-2.4B** | 한국어 특화 검증 — Qwen과 한국어 성능 정면 비교 |
| 3 | **Gemma 4 E4B** | 최신 오픈소스 비교군 — 과제 예시 모델 계열, Apache 2.0 상용 대안 |
| 4 | **Ministral 3 3B** | 최신 경량 다국어 비교군 — Mistral 계열 포함, 긴 컨텍스트와 상용 라이선스 확보 |

#### 파이프라인 B: VLM 단일 모델 (2개)

| 순위 | 모델 | 역할 |
|---|---|---|
| 1 | **Qwen3-VL-4B** | VLM 주력 후보 — Qwen3.5-4B와 동일 조건 비교 가능 |
| 2 | **Gemma 4 E4B** (비전 모드) | 겸용 비교군 — 텍스트/비전 동일 모델로 파이프라인 A/B 비교 |

```
[파이프라인 A] OCR + LLM
  문서 이미지 → OCR 모델 → 추출 텍스트 → 텍스트 LLM → Markdown 결과
  대상: Qwen3.5-4B, EXAONE 3.5-2.4B, Gemma 4 E4B, Ministral 3 3B

[파이프라인 B] VLM 단일 모델
  문서 이미지 → VLM → Markdown 결과
  대상: Qwen3-VL-4B, Gemma 4 E4B(비전)
```

본 프로젝트의 최종 출력 형식은 **Markdown**으로 설정한다.  
Markdown은 JSON 대비 파싱 실패 리스크가 낮고, 사람이 직접 검수(human-in-the-loop)하기에 적합하며,  
VLM의 네이티브 출력 형식과도 일치한다.

비교 시 동일 문서·동일 프롬프트·동일 평가 지표를 적용하여 파이프라인 구조 자체의 장단점까지 분석할 수 있다.
