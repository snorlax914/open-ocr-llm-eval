## 1. 평가지표
### 1.1. 문서 읽기 및 구조 분석 지표

(1) **편집 거리 기반 정확도(CER/WER)**:문자(Character)나 단어(Word) 단위의 오차를 계산하는 방식 -  점수가 낮을수록 성능 굿  

- 공식: Error Rate = (S + D + I) / N, S: 치환(Substitution), D: 삭제(Deletion), I: 삽입(Insertion), N: 전체 글자(또는 단어) 수  

* c는 문자고 w는 단어인데 물류에서는 cer 많이 쓰이고 계약서에서는 WER 인듯  


> +) 소음(노이즈) 견디기:종이가 구겨졌거나, 사진이 흐릿하거나, 낙서가 되어 있어도 글자를 잘 찾아내는지 확인   
> 노이즈의 종류   

> (1)Blur (흐릿함): 사진 초점이 맞지 않는 경우  
> (2)Digits/Signs (낙서): 글자 위에 임의의 숫자나 기호가 겹쳐진 경우  
> (3)Rotation (회전): 문서가 기울어진 경우  
> (4)Salt and Pepper (점잡음): 스캔 품질이 낮아 검은 점들이 박힌 경우  

> 노이즈의 강도(Severity)를 1단계부터 5단계까지 높여가며, 노이즈가 심해질수록 CER(글자 정확도)이 어떻게 변하는지 그래프로 측정  

  
  
(2) **표 및 수식 이해도 (TEDS - Tree Edit Distance-based Similarity)**: 표의 HTML 구조를 트리(Tree)로 변환하여 정답과 얼마나 일치하는지 측정  

- 공식: {TEDS}(T_a, T_b) = 1 - {{EditDist}(T_a, T_b)} over {max(|T_a|, |T_b|)}, T_a, T_b: 정답 표와 예측 표의 트리 구조  


(3) **정확 일치율(Exact Match Rate)**: 이 지표는 OCR 출력 결과 가 실제 데이터와 불일치 없이 정확히 일치하는 문서 내 필드의 비율을 측정  

- 공식:정확한 일치율(EMR) = (올바르게 추출된 필드 수/총 필드 수) * 100  

구조화된 데이터 추출이 필요한 사용 사례에서 OCR 정확도를 평가하는 데 중요한 지표  
송장, 영수증 또는 신분증과 같은 표준화된 양식에서 데이터를 추출할 때 특히 유용  
  
+) **Field-Level Accuracy**:  특정 필드(예: 이름, 날짜, 계좌 번호)를 정확하게 인식해야 하는 정형 및 반정형 문서에 특히 중요  



### 1.2. 내용 활용 및 의미 추출 지표  
(1) **정답유사도(ANLS* : Average Normalized Levenshtein Similarity)**: 생성형 모델이 낸 답이 정답과 완전히 일치하지 않아도 의미적 유사성을 인정해 주는 지표  
 
- 공식: LS^*(x, y) = 1 - {d_t(x, y)}/{l_t(x)}  

재귀적 길이 함수(l_t)를 이용한 구조적 유사도   
보통 0.5 보다 낮으면 오답 처리  

(2) **인간 평가 (Human Score/Likert Scale)**: 번역의 품질이나 자연스러움을 사람이 직접 1~5점 척도로 매기는 방식, 평균으로 최종 점수 - 표에는 0점도 잇긴 한데 걍 기준점(헛소리했을때)충실성(General Quality)유창성(Disource Awareness)기준

### 1.3. 맥락 및 지시 수행 지표 (Context & Instruction)  
(1) **긴 문맥 및 지시 수행 (LIE 스코어: Length-Instruction-Enhanced)**: 입력 길이 증가에 따른 성능 유지력과 긴 문서에서 특정 정보를 찾는 능력과 지시사항 준수 여부를 평가  

- 공식: {Score} = {Match}({Instruction Constraints} x {Metric}({Response, GT})  
- {Score} = {Judge}_{LLM}({Response}, {GT}, {Instructions})  

단순 매칭이 아닌, '글자 수 제한', '형식 지정' 등의 준수여부를 가중치로 반영  

  
(2) **레이아웃 기반 강화학습 보상 (LayoutRL Reward)**: 레이아웃 분석 능력을 정량화하기 위해 사용되는 복합 점수  

- 공식:R = w_1{NED} + w_2  {Acc}_{para} + w_3   {Pres}_{order}, {NED}: 편집 거리, {Acc}_{para}: 단락 개수 정확도, {Pres}_{order}: 읽기 순서 보존도  



## 2. 벤치마크  

(1) **OHRBench**: OCR이 읽은 내용을 바탕으로 인공지능이 질문에 답을 얼마나 잘하는지(RAG 성능) 확인하는 시험   
– ANLS* 답변정확도  
오타가 났을 때 LLM이 얼마나 문맥으로 이를 보정하여 정확한 정보를 찾아내는지(RAG 성능) 확인하기 굿~  

(2) **OmniDocBench**: PDF 문서 안에 있는 복잡한 표, 그림, 수식 등을 얼마나 잘 나누고 읽는지 확인하는 시험   
– TED + EMR  
복잡한 레이아웃과 표 구조 많을테니까 이거 평가하려면 이거 조을듯..  
  
(3) **L-Eval (엘-이벨)**: 아주 긴 글(문서 500개 이상)을 주고 질문을 던져서, 인공지능이 인내심 있게 끝까지 읽고 답하는지 보는 시험  
규정이나 이런 거 넣을 거면~...  
  
(4) **olmOCR Bench**: 실제 문서 이미지에서 글자를 추출하는 실력을 평가
