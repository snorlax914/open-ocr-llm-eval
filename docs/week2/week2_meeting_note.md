## 2. 회의 내용

---

### **1. 모델 선정 방향**

- OCR 모델 비교 검토 진행 (PaddleOCR, EasyOCR 등)
- 정확도뿐 아니라 **처리 속도 중심**으로 평가 필요
- PaddleOCR:
    - 레이아웃 인식 성능 우수
    - 속도 문제 (페이지당 10초 이상)
    - 한글 인식 한계 존재

+)VLM 기반 OCR은 속도 문제로 실서비스 적용 어려움

**-> OCR은 정확도보다 속도 중심으로 재선정 필요**

### **2. LLM 모델 선정 방향**
- OCR 결과 기반 문서 카테고리 분류 수행
- Thinking 모델보다 **Instruct 모델** 사용이 적합
    - Qwen2.5 (7B), Gemma: 응답 속도 느려서 서비스 부적합
- 고려 사항:
    - 응답 속도
    - 불필요한 설명 여부
    - 다국어 처리 성능 (한/영)


### **3. 평가 기준 설정**
- OCR 평가 기준:
    - 한국어 인식 정확도
    - 영어 인식 정확도
    - 레이아웃 인식 능력
    - 처리 속도 (페이지당 시간, 중요)
- LLM 평가 기준:
    - 카테고리 분류 정확도
    - 응답 속도
    - 출력 형식 적합성
- 속도는 실제 테스트 기반으로 측정

### **4. 데이터 및 테스트 환경**
- 테스트 데이터:직접 데이터 수집하여 진행
    - 공공기관 문서, 학교 공지 PDF 등...
- 향후 실제 기업 문서 데이터 활용 예정 (가능 시 별도 공유)

- 실행 환경:
    - Colab 및 로컬 환경 모두 고려
    - GPU 환경별 성능 차이 고려 필요
    - -> Ollama 써서 로컬로도 돌려보고 코랩에서도 돌려보고~.. 속도 차이 등 성능 보기 

### **5. 서비스 구현 방향**
- 문서 업로드 → OCR → LLM 기반 질의응답 구조
- 문서 전처리 X
- 문서 카테고리 자동 분류 기능 포함
- 문서별 채팅 인터페이스 구성

## **3. 결정사항**

---

- OCR 모델은 **속도 중심**으로 선정
- LLM은 **Instruct 모델** 기반으로 선정
- 모델은 직접 실행 후 결과 기반으로 평가
- 서비스 기준 처리 속도: 5초 이내
